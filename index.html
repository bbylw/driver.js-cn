核心特性

不只是引导库

Driver.js 可用于任何需要为页面添加覆盖层的场景——高亮组件、上下文帮助、焦点切换、模态框，以及更多。

极简上手

零外部依赖，几行代码即可创建产品引导。引入、配置、执行——就是这么简单。

超轻量

Gzip 压缩后仅 ~5kb，比同类库（12kb+）小了一半以上，不给项目增加负担。

高度可定制

强大的 API 和丰富的配置选项，从动画速度到弹出框位置，一切尽在掌控。

高亮任意元素

可以高亮页面上的任何 DOM 元素，支持单个元素聚焦或多步骤序列引导。

钩子与回调

为每一步提供生命周期钩子，可在高亮前、高亮后、取消选中时执行自定义逻辑。

跨浏览器兼容

在所有主流浏览器中行为一致，键盘友好，满足无障碍访问需求。

安装

一分钟上手

选择你喜欢的包管理器或通过 CDN 直接引入。

包管理器

$ npm install driver.js
$ pnpm install driver.js
$ yarn add driver.js

安装完成后，在项目中引入模块和样式：

import { driver } from "driver.js";
import "driver.js/dist/driver.css";

CDN 引入

无需安装，直接在 HTML 中添加以下标签：

<!-- 样式表 -->
<link rel="stylesheet"
href="https://cdn.jsdelivr.net/npm/driver.js@latest/dist/driver.css"/>
<!-- JavaScript -->
<script src="https://cdn.jsdelivr.net/npm/driver.js@latest/dist/driver.js.iife.js"></script>

使用 CDN 时，通过全局对象访问：

const driver = window.driver.js.driver;
const driverObj = driver();
快速开始

三步创建产品引导

引入库、定义步骤、启动引导——整个流程就这么简单。

1

引入 Driver.js

通过包管理器或 CDN 引入库文件和样式表。

2

定义引导步骤

创建步骤数组，每一步指定目标元素和弹出框内容。

3

启动引导

调用 drive() 方法开始产品引导之旅。

quickstart.js
import { driver } from "driver.js";
import "driver.js/dist/driver.css";
const driverObj = driver({
showProgress: true,
animate: true,
steps: [
{
element: "#header",
popover: {
title: "导航栏",
description: "这是主导航区域"
}
},
{
element: "#search",
popover: {
title: "搜索",
description: "在这里快速查找内容"
}
},
{
element: "#profile",
popover: {
title: "个人中心",
description: "管理你的账户设置"
}
}
]
});
driverObj.drive();

高亮单个元素

不需要完整的产品引导？使用 highlight() 方法聚焦单个 DOM 元素。

highlight.js
const driverObj = driver();
driverObj.highlight({
element: "#new-feature",
popover: {
title: "新功能：深色模式",
description: "在工具栏中一键切换主题"
}
});
API 参考

完整 API 一览

实例化后返回的对象提供以下方法，用于控制引导流程和查询状态。

引导控制与导航

drive(index?)

启动引导序列，默认从第一步开始，可传入索引指定起始位置。

highlight(options)

高亮单个 DOM 元素，使用 DriveStep 格式配置弹出框。

moveNext()

前进到下一步。

movePrevious()

回退到上一步。

moveTo(index)

跳转到指定步骤。

destroy()

终止并清理当前引导会话。

refresh()

重新计算并重绘高亮区域，适用于布局变化时。

状态查询

isActive()

检查当前是否有引导或高亮正在运行。

hasNextStep()

是否存在下一步。

hasPreviousStep()

是否存在上一步。

getActiveIndex()

获取当前步骤的索引。

getActiveElement()

获取当前步骤关联的 DOM 节点。

动态修改

setConfig(options)

动态更新全局配置。

setSteps(steps[])

动态替换步骤数组。

getConfig() / getState()

获取当前配置或内部状态对象。

配置项

灵活的配置系统

通过配置对象精细控制引导体验的每一个细节。

全局配置

animate boolean

是否启用过渡动画效果。

showProgress boolean

是否显示步骤进度指示器（如 1/5）。

overlayOpacity number

覆盖层的透明度。

smoothScroll boolean

滚动到目标元素时是否使用平滑滚动。

allowClose boolean

是否允许点击覆盖层关闭引导。

allowKeyboardControl boolean

是否允许使用键盘方向键导航。

stagePadding number

高亮区域的内边距。

stageRadius number

高亮区域的圆角半径。

弹出框定位

side

弹出框位置：top right bottom left。空间不足时自动翻转。

align

对齐方式：start center end。

popoverClass

为弹出框添加自定义 CSS 类名。

生命周期钩子

const driverObj = driver({
// 步骤高亮后触发
onHighlighted: (el, step) => {
console.log("已高亮:", el);
},
// 元素即将被高亮
onHighlightStarted: (el, step) => { },
// 元素取消选中
onDeselected: (el, step) => { },
// 引导完成时触发
onDestroyStarted: () => { },
});
